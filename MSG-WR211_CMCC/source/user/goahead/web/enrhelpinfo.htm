Load Balancing

Partial Load Balancing Only when all the primary connections don’t work, the backup connections will start working. Once a primary connection is back to normal, it will automatically switch back to the primary connection.

Connection Detection

The system provides connection detection to monitor connections in real-time. In case a connection fails, traffic is instantly redirected to other available connections to ensure continuation of services.

Basic Settings

You can enable or disable wireless function, configure SSID, wireless mode, etc.
AP Mode In this mode, the Device can connect to other wireless network devices in AP Client mode, and at the same time it can provide connectivity for wireless clients.
APClient Mode In this mode, the Device can connect to a remote network device in AP mode, and at same time it can provide connectivity for wireless clients.
Repeater Mode In this mode, the Device can connect to other wireless network devices in bridge mode, repeater mode or lazy mode, and at the same time it can provide connectivity for wireless clients.
Bridge Mode In this mode, the Device can connect to other wireless network devices in repeater mode or lazy mode. However, wireless clients are unable to connect to the Device directly.
Lazy Mode In this mode, the Device can connect to other wireless network devices in bridge mode or repeater mode, and at the same time it can provide connectivity for wireless clients.
SSID The SSID (Service Set Identification) is used to uniquely identify a wireless network. It is case sensitive.
Wireless Mode The options are 11g Only, 11n Only and 11b/g/n Mixed.
11g Only It allows both 802.11g and 802.11n wireless clients to connect to the Device at 802.11g data rates with a maximum speed of 54Mbps.
11n Only It only allows 802.11n wireless clients to connect to the Device at 802.11n data rates with a maximum speed of 300Mbps.
11b/g/n Mixed It allows 802.11b, 802.11g and 802.11n wireless clients to connect to the Wireless Router at their respective data rates. The maximum speeds are 11Mbps, 54Mbps and 300Mbps respectively.
Channel Select the wireless channel used between the Device and wireless clients. If there are multiple wireless network devices in your area, please make sure that their channels don’t interfere with each other.
Channel Width Specify the range of frequencies for your wireless network. Note that this parameter can only act on 802.11n wireless clients. 802.11b and 802.11g wireless clients can only use 20MHz channel.
Enable SSID Broadcast If you enable this feature, the Device will broadcast its SSID periodically. However, this feature also makes it easier for hackers to know your SSID and break into your WLAN. You'd better disable it to improve security of your WLAN.
AP SSID The SSID of the remote AP.
AP MAC Address The MAC address of the remote AP.
Security Mode The options are None, WPA-PSK/WPA2-PSK, and WEP. When using WPA-PSK or WPA2-PSK, there are two choices for data encryption: TKIP or AES.

Wireless Security Settings

WEP
Authentication Type The options are Auto, Open System and Shared Key.
Auto The Device will automatically choose Open System or Shared Key authentication.
Open System It allows any wireless client to associate with the Device whether that client has the correct WEP key or not. However, the client must have the correct keys to send or receive data from the Device.

Key Format You can enter WEP keys in either Hex format or ASCII format. Hex format (case insensitive) stands for any combination of hexadecimal digits (0-9, a-f, A-F). ASCII format (case sensitive) stands for any combination of ASCII characters.

WPA-PSK/WPA2-PSK

Port-based VLAN

A VLAN (Virtual Local Area Network) is a group of devices that form a logical LAN segment, that is, a broadcast domain. This feature can help simplify network management, enhance network security, and improve network performance.

Port-based VLAN allows you to group the switch ports into multiple VLANs. Ports on the same VLAN can communicate with each other, but they can't communicate with ports on different VLANs.
Note A port can belong to one or more VLANs. By default, all ports are members of VLAN 1 by default.

Link Aggregation

It provides a means to combine multiple switch ports together to form a higher-bandwidth single logical link, known as a Link Aggregation Group (LAG), whose total bandwidth is approximately the sum of the individual switch ports.
Note 1. All ports in a LAG must have the same port settings (except Port Name), port speed and duplex mode, and must belong to the same VLAN.
2. A LAG can’t contain the mirroring port.

Port-based VLAN

A VLAN (Virtual Local Area Network) is a group of devices that form a logical LAN segment, that is, a broadcast domain. This feature can help simplify network management, enhance network security, and improve network performance.

Port-based VLAN allows you to group the switch ports into multiple VLANs. Ports on the same VLAN can communicate with each other, but they can't communicate with ports on different VLANs.
Note A port can belong to one or more VLANs. By default, all ports are members of VLAN 1.

Link Aggregation

It provides a means to combine multiple switch ports together to form a higher-bandwidth single logical link, known as a Link Aggregation Group (LAG), whose total bandwidth is approximately the sum of the individual switch ports.
Note 1. All ports in a LAG must have the same port settings (except Port Name), port speed and duplex mode.
2. All ports in a LAG must belong to the same VLAN, and must have the same Tag VLAN port settings if you enable tag-based VLAN on the switch.
3. A LAG can’t contain the mirroring port.
4. All ports in a LAG must have the same rate limit settings.
5. If LACP or RSTP is enabled on a port, the port cannot be added to a LAG.

Port Mirroring

It copies the traffic from the specified ports to another port where the traffic can be monitored with an external network analyzer. This feature can help you perform traffic monitoring, performance analysis and fault diagnosis.
Mirroring Port You can choose only one mirroring port that will mirror the traffic of the mirrored port(s).

Note 1. The mirroring port cannot be a mirrored port.
2. The mirroring port cannot be a LAG member port, which is displayed in red color in Mirroring Port drop-down list.

System Information
Curent System Time The Device’s current date and time.
System Up Time The elapsed time since the Device was last started.
CPU The current CPU usage.
Memory The current memory usage.
Note The CPU and Memory are displayed as a status bar and percentage value. The color of the status bar indicates the usage percentage for each resource.
1. Blank: Percentage is below 1%.
2. Green: Percentage is between 1% and 50% (below 50%).
3. Orange: Percentage is between 50% and 70% (below 70%).
4. Red: Percentage is equal to or above 70%.
SN The serial number of the Device.
Model The product model of the Device.
Hardware The hardware version of the Device.
Software The software version of the Device.
System Log It allows you to view system logs.
